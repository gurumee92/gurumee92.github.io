1부. 재료 준비: 테트리스 월드의 기초 데이터
01. 안녕, Go!: Go 설치 및 첫 인사 (fmt.Println).

02. 변하는 값과 변하지 않는 규칙: 변수(var, :=)와 상수(const).

03. 데이터의 이름표: 기본 타입과 형 변환(Type Conversion).

보완: 좌표 계산 시 int와 float64 사이의 변환이 왜 필요한지 다룸.

04. 한 줄 기차와 집게: 배열, 슬라이스, 그리고 range.

보완: for i 대신 for index, value := range를 써서 안전하게 배열을 훑는 법.

05. 모눈종이 게임판: 2차원 배열과 격자 시스템.

2부. 기본 동작: 보고, 듣고, 판단하기
06. 도화지에 그리기: 표준 출력과 ANSI 코드.

07. 컴퓨터의 귀: 표준 입력과 키보드 값 읽기.

08. 갈림길에서 판단하기: if 문과 switch 문.

보완: 키보드 입력(a, s, d, w) 처리에 더 깔끔한 switch 활용법.

09. 끝없이 반복하기: for 루프와 게임 엔진의 기초.

3부. 논리의 조립: 효율적인 데이터 관리
10. 기능 묶기와 보고서: 함수(Function)와 다중 반환값.

보완: Go의 특징인 "결과와 에러를 동시에 돌려주기" 맛보기.

11. 부품 조립하기: 구조체(Struct)로 블록 정보 묶기.

12. 보물지도와 주소: 포인터(Pointer)와 메모리의 원리.

13. [중요] 사고 예방: **에러 처리(error)**와 defer.

보완: "문제가 생기면 즉시 보고하기"와 "끝날 때 꼭 정리하기(화면 복구 등)".

4부. 설계의 기술: 고수가 짜는 테트리스 (OOP)
14. 블록 스스로 움직이기: 메서드(Method)와 포인터 리시버.

15. 똑똑한 블록들의 약속: 인터페이스(Interface).

보완: "모든 블록은 '그려질 수 있다'"는 추상적 규칙 세우기.

16. 비밀과 공개: **패키지(Package)**와 가시성(대소문자 규칙).

보완: Game 구조체의 속성을 감추고 필요한 기능만 외부에 공개하기.

5부. 시간의 마법: 멈추지 않는 세상 (Concurrency)
17. 뚝, 뚝, 떨어지는 블록: time.Tick과 시간 제어.

18. 분신술 고루틴: go 키워드로 독립적인 실행 흐름 만들기.

19. 데이터 신호등: **채널(Channel)**과 select.

보완: 여러 입력(키보드 vs 타이머) 중 먼저 온 것을 안전하게 처리하기.

20. 시스템 완성: 리팩토링 및 나만의 테트리스 출시.
